# Staff Check-in + Badges (Milestone 8) Implementation Plan

> **For agentic workers:** Use superpowers:executing-plans. Steps use checkbox (`- [ ]`).

**Goal:** Hybrid check-in (custom browser print-station + pretixSCAN/webhooks) with pretix check-in lists as source of truth, eligibility gating, and 4×6 role badges.

**Architecture:** `lib/checkin/service.ts` validates eligibility (`registrationState === issued`), redeems against the pretix check-in list, logs badge prints + audit. Badge React components print 4×6 via CSS. A `/api/webhooks/pretix` endpoint (shared-secret verified) updates counters. Staff routes gated to check-in staff (assigned events only).

**Tech Stack:** Next.js 16, Prisma, Vitest.

Spec: `docs/superpowers/specs/2026-06-09-checkin-design.md`
