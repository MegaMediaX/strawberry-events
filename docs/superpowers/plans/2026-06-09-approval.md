# Approval + Attendee Flow (Milestone 7) Implementation Plan

> **For agentic workers:** Use superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Event/ticket-level approval that auto-approves, holds for manual review, rejects, or routes to payment — driven by event settings, without breaking the finance/payment flow.

**Architecture:** `AttendeeOrder` gains `approvalStatus` (separate from payment `status`). A derived `registrationState(order)` powers attendee/admin display. Registration resolves `requiresApproval(event,itemIds)`; approval service does approve/reject (guarded, scoped, audited, emailed). Admin approval queue + attendee state pages consume it.

**Tech Stack:** Next.js 16, Prisma, Vitest.

Spec: `docs/superpowers/specs/2026-06-09-approval-design.md`
