# Payments + Finance UI (Milestone 6) Implementation Plan

> **For agentic workers:** Use superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Finance order list + COD mark-paid lifecycle (issue ticket, email, audit, pretix sync) under org isolation, blocked while impersonating.

**Architecture:** `lib/finance/service.ts` owns scoped reads + the mark-paid transition (pretix → DB → email → audit). Auth gains an `impersonating` flag. Admin finance pages render the list/detail and call Server Actions.

**Tech Stack:** Next.js 16, Prisma, Vitest.

Spec: `docs/superpowers/specs/2026-06-09-finance-design.md`
